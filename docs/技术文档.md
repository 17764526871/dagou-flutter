# Dagou AI 技术文档

## 项目概述

Dagou AI 是一个基于 Flutter 开发的端侧多模态 AI 助手应用，使用 Google Gemma 4 E2B 模型，完全运行在移动设备上，无需网络连接。

## 核心模块

### 1. AIService (AI 服务)

**职责**: 管理 Gemma 4 模型的生命周期和推理

**关键方法**:
```dart
class AIService {
  // 初始化模型
  Future<void> initialize() async
  
  // 发送消息（同步）
  Future<String> sendMessage(String text, {Uint8List? imageBytes})
  
  // 发送消息（流式）
  Stream<String> sendMessageStream(String text, {Uint8List? imageBytes})
}
```

**实现细节**:
- 单例模式，全局唯一实例
- 从 assets 加载内置模型
- 支持文本和图片多模态输入
- 提供同步和流式两种响应方式

### 2. LoadingScreen (加载界面)

**职责**: 应用启动时加载模型

**功能**:
- 显示加载动画
- 显示加载进度
- 错误处理和重试
- 完成后自动跳转到聊天界面

### 3. ChatScreen (聊天界面)

**职责**: 主要的用户交互界面

**功能**:
- 消息列表显示
- 多模态输入（文本、图片、语音）
- 实时响应显示
- 语音识别和播报

### 4. MessageBubble (消息气泡)

**职责**: 显示单条消息

**特性**:
- 区分用户和 AI 消息
- 支持文本和图片显示
- 渐变色设计
- 圆角气泡样式

### 5. InputBar (输入栏)

**职责**: 用户输入组件

**功能**:
- 文本输入
- 图片选择按钮
- 拍照按钮
- 视频选择按钮
- 语音输入按钮

## 模型集成

### Gemma 4 E2B 模型

**规格**:
- 参数量: 2B
- 模型大小: ~2.5GB
- 支持模态: 文本、图片、音频
- 上下文长度: 8192 tokens

**集成方式**:
```dart
// 1. 安装模型
final installer = FlutterGemma.installModel(
  modelType: ModelType.gemmaIt,
  fileType: ModelFileType.task,
);

await installer
  .fromAsset('assets/models/gemma-4-E2B-it-web.task')
  .install();

// 2. 创建模型实例
final model = await FlutterGemma.getActiveModel(
  maxTokens: 8192,
  supportImage: true,
  maxNumImages: 1,
  supportAudio: true,
  preferredBackend: PreferredBackend.gpu,
);

// 3. 创建聊天会话
final chat = await model.createChat(
  temperature: 1.0,
  topK: 64,
  topP: 0.95,
  supportImage: true,
  supportAudio: true,
);
```

**推理流程**:
```dart
// 创建消息
final message = Message(
  text: "分析这张图片",
  imageBytes: imageData,
  isUser: true,
);

// 添加到历史
await chat.addQuery(message);

// 获取响应
final response = await chat.generateChatResponse();
if (response is TextResponse) {
  print(response.token);
}
```

## 性能优化

### 1. 模型加载优化
- 模型内置在 assets 中，避免网络下载
- 首次加载后缓存在内存中
- 使用 GPU 加速推理

### 2. 图片处理优化
- 图片自动压缩到合适尺寸
- 使用 JPEG 格式减小体积
- 异步加载避免阻塞 UI

### 3. 内存管理
- 及时释放不用的资源
- 控制消息历史长度
- 图片使用完后释放内存

### 4. UI 性能
- 使用 const 构造函数
- 避免不必要的 rebuild
- 列表使用 ListView.builder

## 故障排查

### 常见问题

**1. 模型加载失败**
- 检查 assets 路径是否正确
- 确认模型文件完整
- 查看内存是否充足

**2. 权限被拒绝**
- 在设置中手动授予权限
- 重新安装应用

**3. 推理速度慢**
- 确认使用 GPU 后端
- 减小 maxTokens 参数
- 关闭其他占用资源的应用

**4. 内存不足**
- 清理消息历史
- 减小图片尺寸
- 重启应用
