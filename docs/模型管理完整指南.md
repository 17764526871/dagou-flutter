# 模型管理完整指南

## 概述

Dagou AI 提供三种方式获取和管理AI模型：
1. **局域网下载** - 从电脑下载模型到手机
2. **本地文件选择** - 从手机存储导入已有模型
3. **内置模型** - 应用自带的 Gemma 4 E2B 模型

## 一、局域网下载（推荐）

### 1.1 在电脑上启动服务器

#### 方式一：使用 Node.js（推荐）

```bash
# 进入项目目录
cd f:\work\github\dagou-flutter

# 启动服务器
node model_server.js
```

服务器启动后会显示：
```
============================================================
🚀 NAS模型HTTP服务器已启动
============================================================
📁 模型目录: U:\DG model
🌐 端口: 8080

可用模型:
  ✅ FastVLM-0.5B.litertlm (1102.8MB)
  ✅ gemma-4-E2B-it.litertlm (2463.4MB)
  ✅ gemma-4-E4B-it.litertlm (3485.2MB)

访问地址:
  http://localhost:8080
  http://192.168.31.50:8080    ← 手机使用这个地址

API接口:
  GET /api/models - 获取模型列表

按 Ctrl+C 停止服务器
============================================================
```

**记下局域网IP地址**（例如：`192.168.31.50:8080`），手机需要用这个地址。

### 1.2 在手机上下载模型

1. 确保手机和电脑连接到**同一个WiFi**
2. 打开 Dagou AI 应用
3. 进入 **设置 → 模型管理**
4. 在"局域网服务器"输入框中输入电脑的IP地址：
   ```
   192.168.31.50:8080
   ```
5. 点击"连接服务器"按钮
6. 连接成功后会显示可下载的模型列表
7. 点击模型旁边的"下载"按钮开始下载
8. 下载完成后，模型会保存在手机外部存储中

### 1.3 下载模型到电脑（可选）

如果电脑上还没有模型文件，可以运行下载脚本：

```bash
# 下载所有可用模型
node download_models.js
```

脚本会尝试从多个镜像源下载模型到 `U:\DG model` 目录。

## 二、本地文件选择

如果你已经有模型文件（从其他渠道下载或复制到手机），可以直接选择：

1. 打开 Dagou AI 应用
2. 进入 **设置 → 模型管理**
3. 点击"从本地文件夹选择模型"卡片
4. 点击"选择"按钮
5. 在文件选择器中找到模型文件（支持 `.task`、`.litertlm`、`.bin` 格式）
6. 选择后应用会自动加载该模型

**支持的文件格式：**
- `.litertlm` - LiteRT LM 格式（推荐）
- `.task` - TensorFlow Lite Task 格式
- `.bin` - 二进制格式

## 三、模型存储位置

### 3.1 电脑端（服务器）

模型存储在：`U:\DG model`

可用模型：
- `gemma-4-E2B-it.litertlm` (2.5GB) - 内置多模态模型
- `gemma-4-E4B-it.litertlm` (3.5GB) - 更大的多模态模型
- `FastVLM-0.5B.litertlm` (1.1GB) - 快速视觉语言模型

### 3.2 手机端

**局域网下载的模型：**
```
/storage/emulated/0/Android/data/com.example.dagou_flutter/files/models/
```

**特点：**
- 保存在应用外部存储
- 卸载应用后**不会丢失**
- 可以通过文件管理器访问

**内置模型：**
```
assets/models/gemma-4-E2B-it.litertlm
```

**特点：**
- 打包在应用内
- 卸载应用后会丢失
- 无法通过文件管理器访问

## 四、服务器脚本说明

### 4.1 model_server.js

**功能：**
- 提供HTTP服务器，分发模型文件
- 自动检测并打印本机IP地址
- 提供Web界面和API接口

**使用：**
```bash
node model_server.js
```

**API接口：**
- `GET /` - Web界面
- `GET /api/models` - 获取模型列表（JSON）
- `GET /<filename>` - 下载模型文件

**配置：**
```javascript
const MODEL_DIR = 'U:\\DG model';  // 模型目录
const PORT = 8080;                  // 服务器端口
```

### 4.2 download_models.js

**功能：**
- 批量下载AI模型到NAS
- 支持多个镜像源（hf-mirror.com、huggingface.co）
- 显示下载进度
- 自动跳过已存在的文件

**使用：**
```bash
node download_models.js
```

**下载的模型：**
- Gemma 3 1B
- FunctionGemma 270M
- DeepSeek R1
- Qwen2.5 0.5B
- Phi-4 Mini
- SmolLM 135M

**注意：** 部分模型可能因为HuggingFace限制而下载失败。

## 五、常见问题

### 5.1 手机无法连接服务器

**检查清单：**
1. 手机和电脑是否连接到同一个WiFi？
2. IP地址是否正确？（不要用 `localhost`）
3. 端口号是否正确？（默认 8080）
4. 电脑防火墙是否阻止了端口？
5. 服务器是否正在运行？

**测试连接：**
在手机浏览器中访问：`http://192.168.31.50:8080`
如果能看到网页，说明连接正常。

### 5.2 下载速度慢

**原因：**
- WiFi信号弱
- 电脑硬盘读取速度慢
- 手机存储空间不足

**解决方案：**
- 靠近WiFi路由器
- 使用5GHz WiFi（如果支持）
- 清理手机存储空间

### 5.3 模型加载失败

**可能原因：**
1. 文件损坏（下载不完整）
2. 文件格式不支持
3. 模型类型识别错误

**解决方案：**
- 重新下载模型
- 确认文件格式正确（`.litertlm`、`.task`、`.bin`）
- 检查文件大小是否正常

### 5.4 模型下载后找不到

**位置：**
```
/storage/emulated/0/Android/data/com.example.dagou_flutter/files/models/
```

使用文件管理器导航到这个路径即可找到。

## 六、技术细节

### 6.1 模型格式

| 格式 | 说明 | 优点 | 缺点 |
|------|------|------|------|
| `.litertlm` | LiteRT LM 格式 | 性能最好，支持多模态 | 文件较大 |
| `.task` | TFLite Task 格式 | 兼容性好 | 功能有限 |
| `.bin` | 二进制格式 | 文件小 | 需要额外配置 |

### 6.2 模型类型识别

应用会根据文件名自动识别模型类型：
- 包含 `gemma` → `ModelType.gemmaIt`
- 包含 `qwen` → `ModelType.qwen`
- 包含 `deepseek` → `ModelType.deepSeek`
- 其他 → `ModelType.general`

### 6.3 网络协议

服务器使用标准HTTP协议，支持：
- CORS跨域访问
- 断点续传（部分支持）
- 大文件传输

## 七、最佳实践

1. **首次使用：** 先用局域网下载一个小模型（FastVLM 0.5B）测试
2. **存储管理：** 定期清理不用的模型，释放空间
3. **网络环境：** 在WiFi环境下下载，避免使用移动数据
4. **备份模型：** 重要模型可以备份到电脑或云盘
5. **服务器安全：** 只在局域网使用，不要暴露到公网
