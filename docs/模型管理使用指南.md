# 模型管理使用指南

## 概述

Dagou AI 支持多种方式管理和加载 AI 模型：
1. **内置模型**：Gemma 4 E2B (2.4GB) 随应用安装
2. **网络下载**：从 HuggingFace 下载其他模型
3. **局域网下载**：从 NAS 或本地服务器快速下载
4. **本地文件**：从手机存储选择已有模型文件

## 功能特性

### ✅ 已修复的问题
- 修复了模型管理页面 UI 溢出问题（能力标签使用 Wrap 自动换行）
- 优化了模型卡片布局，支持多行显示

### 🆕 新增功能
1. **局域网下载**：通过 NAS 或本地 HTTP 服务器快速下载模型
2. **文件选择器**：直接从手机存储选择模型文件
3. **外部存储支持**：模型保存在应用外部存储，卸载应用不会丢失
4. **多种下载方式**：支持 HuggingFace、局域网、本地文件三种方式

## 使用方法

### 1. 内置模型（Gemma 4 E2B）

内置模型随应用安装，无需额外下载：
- 打开"模型管理"页面
- 点击"Gemma 4 E2B"卡片的"切换使用"按钮
- 等待模型加载完成

### 2. 从 HuggingFace 下载

适合首次下载或网络条件良好时使用：

1. 在模型管理页面，找到想要的模型
2. 点击"下载"按钮
3. 选择"从 HuggingFace 下载"
4. 等待下载完成（显示进度条）
5. 下载完成后点击"切换使用"

**注意**：
- 需要稳定的网络连接
- 文件较大（0.5GB ~ 4.3GB）
- 建议在 WiFi 环境下下载

### 3. 从局域网下载

适合多设备使用或已有模型文件服务器时：

#### 在手机上下载

1. 确保手机和服务器在同一局域网
2. 打开模型管理页面
3. 点击"下载"按钮
4. 选择"从局域网下载"
5. 输入模型地址，例如：
   ```
   http://192.168.1.100:8080/models/gemma-4-E4B-it.litertlm
   ```
6. 点击"开始下载"
7. 等待下载完成

**优势**：
- 下载速度快（局域网速度）
- 不消耗外网流量
- 可以多设备共享模型文件

**注意**：需要自行搭建 HTTP 文件服务器，或使用 NAS 的文件共享功能。
- FastVLM-0.5B.litertlm (0.5GB)
- DeepSeek-R1-Distill-Qwen-1.5B.task (1.7GB)
- Qwen3-0.6B.task (586MB)
- Phi-4-mini-instruct.task (3.9GB)
- SmolLM-135M-Instruct.task (135MB)

### 4. 从本地文件选择

如果已经有模型文件（通过其他方式下载）：

1. 在模型管理页面，找到对应的模型
2. 点击"选择文件"按钮
3. 在文件选择器中找到模型文件（.task 或 .litertlm）
4. 选择文件后，点击"切换使用"加载模型

**支持的文件格式**：
- `.task` - 标准 TensorFlow Lite 模型
- `.litertlm` - LiteRT 语言模型格式

## 模型存储位置

### 内置模型
- 位置：`assets/models/gemma-4-E2B-it.litertlm`
- 随应用安装，卸载会删除

### 下载的模型
- 位置：`/storage/emulated/0/Android/data/com.example.dagou_flutter/files/models/`
- 保存在应用外部存储
- **卸载应用不会删除**（需手动清理）

### 用户选择的文件
- 位置：用户选择的任意位置
- 应用只读取，不会移动或复制文件

## 常见问题

### Q: 下载失败怎么办？
A: 
1. 检查网络连接
2. 尝试使用局域网下载
3. 或者在电脑上下载后通过"选择文件"导入

### Q: 模型占用空间太大？
A: 
1. 只下载需要的模型
2. 使用轻量级模型（SmolLM 135M、FunctionGemma 270M）
3. 手动删除不用的模型文件

### Q: 卸载应用后模型还在吗？
A: 
- 内置模型（Gemma 4 E2B）：会删除
- 下载的模型：不会删除，保存在外部存储
- 用户选择的文件：不会删除，保持原位置

### Q: 如何删除已下载的模型？
A: 使用文件管理器删除：
```
/storage/emulated/0/Android/data/com.example.dagou_flutter/files/models/
```

### Q: 局域网下载找不到服务器？
A: 
1. 确保手机和电脑在同一 WiFi
2. 检查防火墙是否阻止了 8080 端口
3. 尝试使用电脑的 IP 地址而不是 localhost

## 技术细节

### 依赖包
- `file_picker: ^8.1.6` - 文件选择器
- `dio: ^5.7.0` - HTTP 下载
- `path_provider: ^2.1.4` - 路径管理
- `path: ^1.9.0` - 路径操作

### 核心服务
- `FilePickerService` - 文件选择服务
- `ModelDownloadService` - 局域网下载服务
- `ModelManager` - 模型管理器（支持外部路径）

### 模型加载优先级
1. 外部路径（用户选择的文件）
2. 内置模型（assets）
3. 网络下载（HuggingFace）

## 开发者说明

### 添加新模型

在 `lib/services/ai/model_manager.dart` 中添加：

```dart
AIModelInfo(
  id: 'model-id',
  name: 'Model Name',
  size: '1.0GB',
  capabilities: ['text', 'image'],
  url: 'https://huggingface.co/...',
  description: 'Model description',
  modelType: ModelType.gemmaIt,
  fileType: ModelFileType.task,
),
```

### 修改 NAS 路径

编辑 `nas_model_server.py`：

```python
MODEL_DIR = r"U:\DG model"  # 修改为你的路径
PORT = 8080  # 修改端口
```
